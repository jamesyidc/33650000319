## ✨ 主要功能

### 1. 数据容错机制 - 二次确认防止API误判

#### 工作流程

```
数据更新检测
  ↓
数据状态判断：有数据 → 无数据？
  ↓ (是)
等待5秒后重新请求API
  ↓
二次确认：仍然无数据？
  ↓ (是)
显示数据丢失警告并记录日志
  ↓ (否)
恢复正常数据显示
```

#### 功能特性

- **自动检测异常**: 当检测到持仓数据从有变无时，自动触发二次确认
- **智能延迟**: 5秒延迟后重新请求API，避免临时故障误判
- **实时提示**: 显示橙色警告提示或绿色成功提示
- **详细日志**: 记录首次检测、二次确认结果、最终决策

#### 应用场景

- ✅ OKX API临时不可用或超时
- ✅ 网络抖动导致请求失败
- ✅ 服务器临时过载返回空数据
- ✅ Python脚本异常退出后重启
- ✅ 持仓真实平仓的正常情况

## 🔧 技术实现

### 核心变量

```javascript
let currentState = null;           // 当前状态
let lastValidState = null;        // 上次有效状态
let isRetryingValidation = false; // 防止重复验证
```

### 关键函数

1. **detectDataLoss(oldState, newState)**
   - 功能：检测数据异常（持仓数量或成本从有变无）
   - 返回：异常账户信息对象或null

2. **showDataWarning(message)**
   - 功能：显示橙色警告提示
   - 参数：警告消息文本
   - 效果：10秒后自动消失

3. **showDataSuccess(message)**
   - 功能：显示绿色成功提示
   - 参数：成功消息文本
   - 效果：5秒后自动消失

4. **loadState() - 增强版**
   - 原功能：加载当前状态
   - 新增：数据异常检测 + 二次确认逻辑
   - 流程：检测 → 警告 → 延迟 → 重试 → 更新

### 异常情况1: API临时故障

3. 更新状态为空持仓
