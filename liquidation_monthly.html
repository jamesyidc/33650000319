✅ 标记持久化修复 - 重新部署后不会丢失 (v2.1 - 2026-02-28)

【问题】重新部署后，设置的多空标记会消失

【根本原因】

  • ❌ API路由定义在 if __name__ == '__main__' 块内部
  • ❌ 通过PM2运行时，main块不执行，路由未注册
  • ❌ 前端调用API返回404，无法保存到服务器
  • ❌ 标记只保存在浏览器localStorage，清除缓存会丢失
  • ❌ 跨浏览器/设备不同步，URL变化后丢失

✅ 修复方案：

  1. 将 /api/liquidation/marks 路由移到全局作用域（main块之前）
  2. 修复GET接口，正确提取marks数组（兼容新旧格式）
  3. 前端实现双重保存：localStorage（本地备份） + 服务器（永久存储）
  4. 前端加载时优先从服务器读取，失败时回退到localStorage
  5. 数据保存在 data/liquidation_marks.json，包含时间戳

🎯 工作流程：

用户点击标记点 → 设置多/空标记
↓
saveMarksToStorage() 函数触发
├─ 保存到 localStorage（即时生效）
└─ 保存到服务器 POST /api/liquidation/marks
└─ 写入 data/liquidation_marks.json

页面加载时 → loadMarksFromStorage()
├─ 优先：GET /api/liquidation/marks（服务器）
└─ 回退：localStorage（本地备份）

📊 数据格式：

{
  "marks": [
    {"time": "2026-02-28 10:00", "markType": "long"},
    {"time": "2026-02-28 12:00", "markType": "short"}
  ],
  "last_updated": "2026-02-28T12:43:11.478999",
  "count": 2
}

✅ 修复后的优势：

  • ✅ 标记永久保存在服务器，重新部署后不会丢失
  • ✅ 跨浏览器/设备同步，所有地方看到相同的标记
  • ✅ 清除浏览器缓存不影响服务器数据
  • ✅ URL变化（sandbox ID变化）不影响标记
  • ✅ 双重保存机制，更安全可靠

💡 使用提示：
1. 标记会自动保存到服务器，无需任何操作
2. 每次设置标记后，控制台会显示 "✅ 已保存 N 个标记到服务器"
3. 页面加载时会显示 "✅ 从服务器加载了 N 个标记"
4. 如果API保存失败，标记仍会保存在localStorage作为备份
5. 详细文档：FIX_LIQUIDATION_MARKS_PERSISTENCE_2026-02-28.md
