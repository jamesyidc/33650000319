## 🔍 截图显示差异原因分析

截图显示的 "260.2s MB" 可能是：

### 1. 单位换算问题
不同系统使用不同的 MB 定义：
- **二进制 (MiB)**: 1 MB = 1024 × 1024 bytes = 1,048,576 bytes
- **十进制 (MB)**: 1 MB = 1000 × 1000 bytes = 1,000,000 bytes

如果使用不同单位:
- 299,066,047 bytes ÷ 1,048,576 = **285.21 MiB**
- 299,066,047 bytes ÷ 1,000,000 = **299.07 MB**

但都不等于 260.2 MB

### 2. 可能的显示错误
- 网页缓存导致显示旧数据
- JavaScript 数值计算精度问题
- API 返回了错误的大小信息

### 3. 不同的备份源
- 截图可能显示的是其他位置的备份
- 或者是上传到云端后压缩的大小
