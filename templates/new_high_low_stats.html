🔄 系统运行逻辑

1. 数据采集层（Backend Collector）

  • PM2守护进程 new-high-low-collector 每3分钟执行一次
  • 读取价格位置数据源: /data/price_position/price_position_YYYYMMDD.jsonl
  • 提取最新快照的28个币种价格
  • 逐个币种比较当前价格与历史最高价/最低价
  • 检测到创新高/低时:
    • 记录事件到JSONL: /data/new_high_low/new_high_low_events_YYYYMMDD.jsonl
    • 更新状态文件: /data/new_high_low/coin_highs_lows_state.json
    • 打印日志: 🔥 创新高 / ❄️ 创新低

2. API接口层（Flask Backend）

  • 路由: GET /api/price-position/new-high-low-stats
  • 读取状态文件获取当前最高/低价
  • 读取过去8天的事件JSONL文件
  • 按时间范围统计: 今天、昨天、3天、7天
  • 返回JSON响应（<50ms）
  • 添加防缓存响应头（Cache-Control: no-store）

3. 前端展示层（HTML + ECharts）

  • 页面路由: /new-high-low-stats
  • 每60秒自动调用API刷新数据
  • 渲染4个统计卡片、2个ECharts图表、2个数据表格
  • 计算距今时长（getTimeElapsed函数）
  • 纯展示逻辑，零业务计算

📄 JSONL文件格式详解

1. 状态文件: coin_highs_lows_state.json

路径: /home/user/webapp/data/new_high_low/coin_highs_lows_state.json

{
  "BTC": {
    "highest_price": 68589.1,           // 历史最高价
    "highest_time": "2026-02-16 22:22:26",  // 创建最高价的时间
    "lowest_price": 68409.8,            // 历史最低价
    "lowest_time": "2026-02-16 22:19:01"    // 创建最低价的时间
  },
  "ETH": { ... },
  ... (共28个币种)
}

2. 事件日志: new_high_low_events_YYYYMMDD.jsonl

路径: /home/user/webapp/data/new_high_low/new_high_low_events_20260216.jsonl

格式: 每行一个JSON对象（JSONL格式，便于追加和流式读取）

// 创新高事件
{
  "time": "2026-02-16 22:22:26",       // 事件发生时间（北京时间）
  "timestamp": 1771251746.0,           // Unix时间戳
  "symbol": "BTC",                     // 币种代码
  "event_type": "new_high",            // 事件类型: new_high | new_low
  "price": 68589.1,                    // 当前价格（创新高）
  "previous_high": 68500.0,            // 前一次最高价
  "previous_time": "2026-02-16 20:15:00"  // 前高创建时间
}

// 创新低事件
{
  "time": "2026-02-16 22:29:24",
  "timestamp": 1771252164.0,
  "symbol": "TAO",
  "event_type": "new_low",             // 创新低
  "price": 188.1,                      // 当前价格（创新低）
  "previous_low": 189.9,               // 前一次最低价
  "previous_time": "2026-02-16 22:25:50"  // 前低创建时间
}

3. 源数据: price_position_YYYYMMDD.jsonl

路径: /home/user/webapp/data/price_position/price_position_20260216.jsonl

由 price-position-collector 每5分钟采集OKX API生成

{

    },
    ... (共28个币种)
  ]
}
⚙️ 核心功能详解

功能1: 创新高/低检测

  • 触发条件:
    • 创新高: current_price > coin_state[symbol].highest_price
    • 创新低: current_price < coin_state[symbol].lowest_price
  • 执行操作:
    • 记录事件到JSONL（时间、币种、价格、前值）
    • 更新状态文件中的最高/低价和时间
    • 打印日志到控制台（PM2日志）
  • 状态持久化: 每次采集后保存状态文件，确保服务重启后数据不丢失

功能2: 多时段统计

  • 今天: 过去24小时的事件（now - 1天 ~ now）
  • 昨天: 24-48小时前的事件（now - 2天 ~ now - 1天）
  • 最近3天: 过去72小时的事件（now - 3天 ~ now）
  • 最近7天: 过去168小时的事件（now - 7天 ~ now）
  • 使用北京时区（pytz.timezone('Asia/Shanghai')）

功能3: 按币种统计

  • 为每个时段计算 by_symbol 字典
  • 统计每个币种的创新高次数和创新低次数
  • 用于生成Top 15币种图表

功能4: 实时距今时长计算

  • 前端JavaScript函数 getTimeElapsed()
  • 计算最高/低价创建时间距当前的时长
  • 显示格式: "X天X小时前" / "X小时X分钟前" / "X分钟前" / "刚刚"
  • 每60秒随页面刷新自动更新

功能5: 防浏览器缓存

  • API响应头: Cache-Control: no-store, no-cache, must-revalidate
  • 前端请求: URL添加时间戳 ?_t=Date.now()
  • Fetch选项: cache: 'no-store'
  • 确保数据永远是最新的，不受浏览器缓存影响
