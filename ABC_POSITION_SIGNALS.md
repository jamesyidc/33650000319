# 📊 ABC Position 信号标记功能说明 v2.0

## 功能概述

在ABC Position页面的图表中，系统会自动计算并标记两种重要信号：
1. **4小时最高点** - 金色星星标记 ⭐
2. **平仓信号** - 红色旗帜标记 🚩

## ⚡ 版本更新 v2.0 (2026-03-19)

### 算法优化
- **旧算法**: 滑动窗口，每个点都判断是否为4小时最高点 → 导致过多标记
- **新算法**: 固定窗口，每4小时只标记一个最高点 → 清晰明确

## 信号算法

### 1. 4小时最高点（⭐金色菱形）

**定义**：将数据按4小时（240分钟）划分为多个窗口，在每个窗口内找出盈亏百分比(pnl_pct)最高的点。

**计算逻辑**：
```
数据时间线: [0分钟 ────────── 240分钟 ────────── 480分钟 ────────── 720分钟 ...]
窗口划分:   [  窗口1: 0-240   ] [  窗口2: 240-480 ] [  窗口3: 480-720 ]

每个窗口只标记一个最高点：
- 窗口1 (0-240分钟): 找出盈亏最高的点 → 标记为最高点⭐
- 窗口2 (240-480分钟): 找出盈亏最高的点 → 标记为最高点⭐
- 窗口3 (480-720分钟): 找出盈亏最高的点 → 标记为最高点⭐
...

算法伪代码:
windowStart = 0
while windowStart < 数据长度:
    windowEnd = min(windowStart + 240, 数据长度)
    windowData = 数据[windowStart:windowEnd]
    
    找出 windowData 中 pnl_pct 最大且 > 0 的点
    如果找到 → 标记为"4小时最高点"
    
    windowStart += 240  // 移动到下一个4小时窗口
```

**标记条件**：
- ✅ 在该4小时窗口内是最高盈利点
- ✅ 盈利百分比 > 0（只标记盈利状态）
- ✅ **每个窗口只有一个最高点**（关键特性）

**图表显示**：
- 符号：⭐（金色菱形 diamond）
- 大小：16px
- 标签："⭐4H高"
- 颜色：金黄色 (#fbbf24)

---

### 2. 平仓信号（🚩红色旗帜）

**定义**：4小时最高点出现后，如果在接下来的2.5小时（150分钟）内没有出现更高的盈利点，则触发平仓信号。

**计算逻辑**：
```
时间线:    最高点⭐ ──────────────── 150分钟后 🚩
           |                        |
检查范围:   ←────── 2.5小时 ────────→

对于每个4小时最高点 hp：
  1. 记录最高点的 pnl_pct 和位置 idx
  2. 定义检查范围: [idx+1, idx+150]
  3. 遍历检查范围内的所有数据点
  4. 如果发现任何 pnl_pct > hp.pnl
     → 不触发平仓信号
  5. 如果整个范围内都没有更高的点
     → 在 idx+150 的位置标记"平仓信号"
```

**标记条件**：
- ✅ 已经标记了4小时最高点
- ✅ 从最高点往后2.5小时内没有更高的点
- ✅ 至少有150分钟的后续数据

**图表显示**：
- 符号：🚩（红色旗帜）
- 大小：14px
- 标签："🚩平仓"
- 颜色：红色 (#ef4444)

## API接口

### 获取信号数据
```bash
GET /abc-position/api/signals?date=20260319&account=A
```

**参数**：
- `date` (可选): 日期，格式 YYYYMMDD，默认今天
- `account` (可选): 账户ID (A/B/C/D)，默认返回所有账户

**响应示例**：
```json
{
  "success": true,
  "data": [
    {
      "account": "A",
      "type": "high_point",
      "timestamp": "2026-03-19T10:30:00+08:00",
      "pnl": 5.67,
      "idx": 450
    },
    {
      "account": "A",
      "type": "close_signal",
      "timestamp": "2026-03-19T13:00:00+08:00",
      "pnl": 4.23,
      "idx": 600,
      "high_point_idx": 450,
      "high_point_pnl": 5.67
    }
  ],
  "date": "20260319"
}
```

### 保存信号数据
```bash
POST /abc-position/api/signals
Content-Type: application/json

{
  "signals": [
    {
      "account": "A",
      "type": "high_point",
      "timestamp": "2026-03-19T10:30:00+08:00",
      "pnl": 5.67,
      "idx": 450
    }
  ]
}
```

## 技术细节

### 时间窗口
- **4小时窗口**: 240分钟，用于标记最高点
- **2.5小时窗口**: 150分钟，用于判断平仓信号
- **时区**: 北京时间 (UTC+8)

### 数据采集频率
- **采集间隔**: 每60秒一次
- **每4小时窗口**: 约240个数据点
- **每天数据点**: 约1440个数据点

### 性能优化
- 前端计算，不依赖后端实时计算
- 信号数据按日期分文件存储
- 使用索引快速定位数据点

## 常见问题 FAQ

### Q1: 为什么有些窗口没有最高点标记？
**A**: 因为该4小时窗口内所有数据点都是亏损的（pnl_pct ≤ 0），只有盈利的点才会被标记。

### Q2: 为什么有些最高点没有平仓信号？
**A**: 可能原因：
1. 数据还不够，没有满150分钟的后续数据
2. 在150分钟内创出了新高，不触发平仓

### Q3: 平仓信号一定要执行平仓吗？
**A**: 不一定。这只是一个参考信号，实际操作需要结合：
- 市场趋势
- 其他技术指标
- 个人风险偏好
- 账户整体情况

### Q4: 如何查看历史信号？
**A**: 使用以下命令查看JSONL文件：
```bash
cat /home/user/webapp/data/abc_position_signals/signals_20260319.jsonl
```

### Q5: 每个窗口一定只有一个最高点吗？
**A**: 是的！这是v2.0的核心改进，每个4小时窗口只标记一个最高点，避免过多标记。
